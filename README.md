# SQL Metadata Parser v4.0

Профессиональный инструмент для анализа SQL-запросов, извлечения метаданных таблиц и колонок с поддержкой различных диалектов SQL. Приложение предоставляет графический интерфейс на PyQt6 для удобной визуализации результатов.

## Возможности

- **Автоматический парсинг SQL-запросов** с использованием библиотеки sqlglot
- **Поддержка диалектов SQL**: Oracle, PostgreSQL, MySQL, SQL Server, Snowflake
- **Извлечение метаданных**:
  - Таблицы и их алиасы
  - Колонки с контекстом использования (SELECT, WHERE, JOIN, ORDER BY и т.д.)
  - Алиасы колонок и таблиц
  - Расчётные колонки (агрегации, выражения)
  - Линейные зависимости (lineage)
- **Визуализация результатов** через древовидные виджеты и вкладки
- **Визуализация графа зависимостей** — построение ориентированного графа колонок с использованием NetworkX и Matplotlib
- **Экспорт данных** в Excel (.xlsx) и JSON форматы
- **Подсветка синтаксиса SQL** в редакторе запросов
- **Загрузка SQL из файлов** и примеры запросов
- **Фильтрация и поиск** по результатам анализа

## Архитектура проекта (после рефакторинга)

Проект был рефакторирован для оптимизации структуры, уменьшения количества файлов и улучшения архитектуры:

### Основные модули

1. **`app.py`** — точка входа приложения
2. **`core/sql_parser.py`** — основной парсер SQL, включает:
   - `SQLDialect` (enum диалектов)
   - `SQLPreprocessor` (препроцессор SQL)
   - `ParserStrategy` (абстрактный класс стратегии парсинга)
   - `SQLGlotParserStrategy` (реализация на sqlglot)
   - `DetailedColumnAnalyzer` (детальный анализатор колонок)
   - `ParserFactory` (фабрика парсеров)
3. **`core/export_manager.py`** — менеджер экспорта данных:
   - `ExportStrategy` (абстрактный класс)
   - `ExcelExportStrategy`, `JSONExportStrategy` (конкретные реализации)
   - `ExportManager` (управление экспортом)
4. **`models/sql_metadata.py`** — модели данных:
   - `SQLMetadata` (основной контейнер метаданных)
   - `TableInfo` (информация о таблице)
   - `ColumnMetadata` (информация о колонке)
5. **`ui/main_window.py`** — главное окно PyQt6, включает:
   - `MainWindow` (основной класс интерфейса)
   - `SQLHighlighter` (подсветка синтаксиса SQL, встроенный класс)
   - `ParseWorker` (рабочий поток для парсинга)
6. **`ui/help_text.py`** — текст справки приложения (вынесен отдельно)
7. **`tests/`** — модульные тесты

### Удалённые/объединённые модули

- **`ui/sql_highlighter.py`** → объединён с `ui/main_window.py`
- **`utils/sql_preprocessor.py`** → объединён с `core/sql_parser.py`

### Ключевые улучшения архитектуры

1. **Уменьшение количества файлов** с 10+ до 8 основных модулей
2. **Логическая группировка** связанных классов (препроцессор и парсер)
3. **Устранение циклических зависимостей** за счёт правильного размещения импортов
4. **Вынос констант** (текст справки) в отдельный модуль для улучшения читаемости
5. **Сохранение принципов SOLID**:
   - Стратегия для парсинга и экспорта
   - Фабрика для создания парсеров
   - Отделение моделей данных от бизнес-логики

## Установка и запуск

### Требования

- Python 3.10+
- Зависимости указаны в `requirements.txt`

### Установка зависимостей

```bash
pip install -r requirements.txt
```

### Запуск приложения

```bash
python app.py
```

### Запуск тестов

```bash
pytest tests/
```

### Демонстрационный скрипт

```bash
python test_usage.py
```

## Использование

1. Запустите приложение `python app.py`
2. Введите SQL-запрос в левое текстовое поле
3. Выберите диалект SQL (по умолчанию Oracle)
4. Нажмите кнопку "Анализировать SQL"
5. Просмотрите результаты во вкладках справа:
   - **Таблицы** — список таблиц с алиасами и типами
   - **Колонки** — детальная информация о колонках с контекстом использования
   - **Линейные зависимости** — связи между таблицами и колонками
   - **Граф зависимостей** — визуализация зависимостей колонок в виде ориентированного графа (NetworkX + Matplotlib)
   - **Статистика** — сводная информация о запросе
   - **Текстовый вывод** — форматированный текстовый отчёт
6. Экспортируйте результаты в Excel или JSON через меню "Экспорт"

## Примеры SQL-запросов

В приложении доступны примеры сложных запросов через меню "Примеры":
- Запрос с JOIN, подзапросами и CTE
- Запрос с оконными функциями и агрегациями
- Запрос с расчётными колонками и алиасами
