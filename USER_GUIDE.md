# Руководство пользователя SQL Metadata Parser v4.0

## Введение

SQL Metadata Parser — это инструмент для анализа SQL запросов, который автоматически извлекает метаданные: таблицы, колонки, алиасы, контексты использования и зависимости. Приложение имеет графический интерфейс на PyQt6, поддерживает несколько диалектов SQL (Oracle, PostgreSQL, MySQL, SQL Server, Snowflake) и предоставляет удобные средства для экспорта результатов.

## Установка и запуск

### Требования
- Python 3.8+
- Установленные зависимости из `requirements.txt`

### Установка
1. Клонируйте репозиторий или скачайте исходный код.
2. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

### Запуск
Запустите главный скрипт:
```bash
python app.py
```

После запуска откроется главное окно приложения.

## Сборка исполняемого файла (EXE)

Для создания автономного исполняемого файла (EXE) используйте средства, расположенные в папке `for_exe`. Это позволяет запускать приложение без установки Python и зависимостей.

### Автоматическая сборка (Windows)

1. Перейдите в папку `for_exe`:
   ```bash
   cd for_exe
   ```
2. Запустите скрипт `build_exe.bat`:
   ```bash
   build_exe.bat
   ```
   Скрипт выполнит следующие действия:
   - Создаст виртуальное окружение (если отсутствует)
   - Установит все необходимые пакеты (PyInstaller, sqlglot, pandas, openpyxl, PyQt6)
   - Выполнит сборку с использованием spec-файла `SQLparser_hook.spec`
   - Скопирует готовый `SQLparser.exe` в корень проекта

3. После успешной сборки вы найдёте `SQLparser.exe` в корневой папке проекта. Запускайте его двойным щелчком.

### Ручная сборка (любая ОС)

Если вы предпочитаете ручную сборку или работаете не под Windows:

1. Установите PyInstaller:
   ```bash
   pip install pyinstaller
   ```
2. Перейдите в папку `for_exe` и выполните команду:
   ```bash
   pyinstaller --clean --log-level INFO SQLparser_hook.spec
   ```
3. Исполняемый файл появится в подпапке `dist`. Вы можете скопировать его в удобное место.

### Примечания

- Размер EXE-файла составляет около 140–150 МБ из-за включения библиотек PyQt6 и pandas.
- По умолчанию EXE запускается с консольным окном (для отладки). Чтобы скрыть консоль, отредактируйте параметр `console=True` на `console=False` в файле `SQLparser_hook.spec`.
- Сборка использует исправленные хуки для PyQt6 и sqlglot, что гарантирует корректную работу.

## Интерфейс приложения

Главное окно разделено на две основные панели:

### Левая панель – ввод SQL
- **Поле ввода SQL**: Вставьте или введите SQL запрос.
- **Кнопки**:
  - **Пример SQL**: Загружает пример сложного запроса.
  - **Загрузить**: Загружает SQL из файла (`.sql`, `.txt`).
  - **Очистить**: Очищает поле ввода и результаты.
- **Переключатель подсветки синтаксиса**: Включает/выключает цветовую подсветку ключевых слов, строк, чисел и комментариев.
- **Выбор диалекта**: Выберите диалект SQL (Oracle, PostgreSQL, MySQL, SQL Server, Snowflake).
- **Кнопка «Анализировать SQL»**: Запускает анализ.

### Правая панель – результаты парсинга
- **Вкладки**:
  - **Таблицы**: Список уникальных таблиц с алиасами, типом и количеством колонок.
  - **Колонки**: Детальная информация о каждой колонке (полное имя, таблица, алиасы, места использования, количество упоминаний).
  - **Линедж**: Зависимости колонок и их использование.
  - **Статистика**: Сводная статистика по запросу.
  - **Текстовый вывод**: Текстовое представление результатов.
- **Общий фильтр**: Поле для фильтрации строк во всех вкладках по введённому тексту.

### Верхняя панель
- **Чекбокс «Тёмная тема»**: Переключает между светлой и тёмной темой интерфейса.
- **Кнопка «Справка»**: Открывает это руководство.

### Нижняя панель
- **Строка состояния**: Отображает текущий статус, количество колонок, таблиц и ошибок.
- **Информационная метка**: Подсказки и сообщения о ходе работы.

## Примеры использования

### 1. Быстрый старт
1. Запустите приложение.
2. В левой панели вставьте SQL запрос, например:
   ```sql
   SELECT u.id, u.name, d.department_name
   FROM users u
   JOIN departments d ON u.dept_id = d.id
   WHERE u.active = 1;
   ```
3. Убедитесь, что выбран правильный диалект (по умолчанию Oracle).
4. Нажмите кнопку **«Анализировать SQL»** (или используйте горячую клавишу `Ctrl+Enter`).
5. Результаты появятся на правой панели. Переключайтесь между вкладками для просмотра деталей.

### 2. Анализ сложного запроса
Используйте кнопку **«Пример SQL»**, чтобы загрузить готовый сложный запрос с CTE, подзапросами, алиасами и агрегатными функциями. Это демонстрирует возможности парсера:
- Определение вычисляемых колонок (COUNT, SUM, AVG).
- Обнаружение алиасов таблиц и колонок.
- Построение линеджа зависимостей.

### 3. Фильтрация результатов
В поле **«Общий фильтр»** введите текст (например, имя таблицы или колонки). Строки во всех деревьях будут отфильтрованы в реальном времени.

### 4. Экспорт результатов
После анализа вы можете экспортировать данные в нескольких форматах:
- **CSV** – табличное представление колонок.
- **Excel** – многостраничный файл с листами «Таблицы», «Колонки», «Линедж», «Статистика», «Текстовый вывод», «Исходный SQL».
- **JSON** – структурированные данные для программной обработки.
- **Текстовый файл** – текстовый вывод результатов.

Для экспорта используйте горячую клавишу `Ctrl+S` (экспорт в CSV) или выберите нужный формат через меню (если реализовано). Приложение предложит выбрать путь сохранения и после успешного экспорта может автоматически открыть файл.

### 5. Копирование в буфер обмена
Нажмите кнопку **«Копировать»** (если есть) или используйте соответствующую функцию, чтобы скопировать сводку колонок в буфер обмена в формате TSV (табулированные данные). Это удобно для вставки в Excel или другие табличные редакторы.

## Горячие клавиши

- `Ctrl+Enter` или `Ctrl+Return` – запустить анализ SQL.
- `Ctrl+S` – экспортировать результаты в CSV.
- (Дополнительные горячие клавиши могут быть добавлены в будущих версиях.)

## Часто задаваемые вопросы

### Какие диалекты SQL поддерживаются?
Поддерживаются Oracle, PostgreSQL, MySQL, SQL Server, Snowflake. Выбор диалекта влияет на парсинг специфичных конструкций.

### Может ли парсер обрабатывать вложенные подзапросы и CTE?
Да, парсер корректно обрабатывает Common Table Expressions (CTE), подзапросы в FROM/JOIN/WHERE, определяет их как таблицы типа «Подзапрос» или «CTE».

### Как интерпретируются вычисляемые колонки?
Колонки, являющиеся результатом вызова функций (COUNT, SUM, AVG и др.) или арифметических выражений, помечаются флагом `is_calculation`. Для них указывается тип вычисления (aggregate, function, arithmetic) и исходное выражение.

### Что делать, если парсер выдаёт ошибку?
1. Убедитесь, что SQL синтаксически корректен для выбранного диалекта.
2. Проверьте, нет ли неподдерживаемых конструкций (например, специфичных функций).
3. Если ошибка повторяется, сообщите о проблеме, приложив текст запроса и выбранный диалект.

### Можно ли анализировать несколько запросов одновременно?
Нет, в текущей версии анализируется один запрос за раз. Однако вы можете объединить несколько запросов в один скрипт (например, через UNION) или анализировать их поочерёдно.

### Как добавить поддержку нового диалекта SQL?
Требуется расширить перечисление `SQLDialect` в `core/sql_dialect.py` и добавить соответствующее преобразование в функцию `dialect_to_sqlglot`. Также может потребоваться адаптация препроцессора (`SQLPreprocessor`) для обработки специфичных конструкций.
